Material Button Card 🛎️ ​

The material-button-card is a super versatile card 🃏 that can be used to display and control a variety of entities in Home Assistant. It can be configured to act as a switch 💡, a button 🔘, a thermometer 🌡️, and much more.

🌙 Dark Mode

Configuration 🛠️ ​

Here are the configuration options for the material-button-card:

NameTypeDescriptionDefault
typestringcustom:material-button-cardRequired
control_typestringThe type of control the button will have. See Control Types for more information.generic
namestringThe name to display on the card. 📛
entitystringThe entity ID to associate with the card. 🆔
use_default_iconbooleanUse the default icon based on the entity's state. 🖼️true
iconstringA custom icon to display. ✨
dual_iconbooleanUse two different icons for the "on" and "off" states. 🎭false
icon_onstringThe icon to display when the entity is "on". ✅
icon_offstringThe icon to display when the entity is "off". ❌
use_default_textbooleanUse the default "on" and "off" text. 🔡true
text_onstringCustom text to display when the entity is "on". 켜짐
text_offstringCustom text to display when the entity is "off". 꺼짐
use_material_colorboolean(Only for thermometer) Use different colors based on the state. 🎨false
fix_temperaturestring(Only for thermometer) Set to true or auto to fix temperature display issues. 🔧false
use_default_togglebooleanEnable automatic actions based on entity state. If false, you can configure tap_action and hold_action. 🔄true
tap_actionobjectAction to perform on tap. See Actions for more information. 👇
hold_actionobjectAction to perform on hold. See Actions for more information. HOLD

Control Types 🎛️ ​

The control_type option determines the behavior and appearance of the card. Here are the possible values:

  • generic: For generic entities like switches, lights, and input booleans. সুইচ
  • thermometer: For climate entities. It will display the temperature and allow you to control the HVAC. 🌡️
  • automation: For automation entities. It will show the automation's state and allow you to trigger it. 🤖
  • scene: For scene entities. Tapping the card will activate the scene. 🎬
  • media_player: For media player entities. It will show the media player's state and provide playback controls. ⏯️
  • state: Displays the state of an entity without any action. ℹ️
  • action: Performs a specific action when tapped. ⚡
  • app_version: Displays the version of the Material Components. 📱

Actions 🚀 ​

You can configure custom actions for tap_action and hold_action.

The action object can have the following properties:

  • action: The action to perform. Possible values are:
    • toggle: Toggles the entity's state. 🔄
    • more-info: Shows the entity's more-info dialog. ℹ️
    • navigate: Navigates to a specific path. 🗺️
    • url: Opens a URL. 🌐
    • none: No action. 🚫
  • navigation_path: The path to navigate to when action is navigate.
  • url_path: The URL to open when action is url.

Examples 📜 ​

Simple Switch 💡 ​

yaml
type: custom:material-button-card
control_type: generic
entity: switch.living_room_light
name: Living Room Light

Thermometer with Custom Icons 🌡️ ​

yaml
type: custom:material-button-card
control_type: thermometer
entity: climate.living_room
name: Living Room
use_default_icon: false
dual_icon: true
icon_on: mdi:hvac
icon_off: mdi:hvac-off

Automation Trigger 🤖 ​

yaml
type: custom:material-button-card
control_type: automation
entity: automation.turn_on_lights_at_sunset
name: Sunset Lights

Scene Activation 🎬 ​

yaml
type: custom:material-button-card
control_type: scene
entity: scene.movie_time
name: Movie Time
icon: mdi:movie

Media Player Controls ⏯️ ​

yaml
type: custom:material-button-card
control_type: media_player
entity: media_player.living_room_tv
name: Living Room TV

Navigation Button 🗺️ ​

yaml
type: custom:material-button-card
control_type: action
name: Go to Lights
icon: mdi:lightbulb
use_default_toggle: false
tap_action:
  action: navigate
  navigation_path: /lovelace/lights
